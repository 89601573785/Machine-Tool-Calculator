# Онлайн-сборщик производственного цеха

**Тема дипломного проекта:** разработка веб-решения создания и конфигурирования производственной линии ООО «ЛЕСКОМ».

Веб-приложение для планирования и визуализации компоновки лесопильной линии: каталог станков, подача бревна, конвейеры на связях, комплексы и лесопильные линии, расчёт линии и PDF-отчёт. Встроено в сайт ЛЕСКОМ (iframe + API).

## Возможности

- **Каталог** с вкладками: все / станки / подача бревна / конвейеры (справочник) / **комплексы и линии**; поиск; drag-and-drop
- **Комплексы и лесопильные линии** — в карточке указаны состав станков; перетаскивание на поле **разворачивает линию** в отдельные станки со связями и конвейерами
- **Станки** на рабочей области с габаритами из каталога, сеткой и привязкой; рабочая зона **26000×18000** px
- **Подача бревна** — накопитель, кантователь, бревнотаска, поштучная выдача, центрователь (отдельная вкладка, ставятся на поле)
- **Связи** между станками с выбором конвейера или **«Без конвейера»**; маршруты разводятся, чтобы линии не сливались при разветвлениях
- Режим **«Связать»** и **Shift + клик** по двум станкам
- Зум, панорамирование, адаптивный шаг сетки
- **Автосохранение** в личный кабинет (~0,9 с после правки); **адаптивная синхронизация** с сервером (~24 с в одиночку, ~4,5 с при совместном редактировании)
- **Выделение области** (рамка на пустом поле), групповое перемещение и удаление; во время перетаскивания подгрузка с сервера откладывается
- При обновлении с сервера — **бесшовное** применение правок; **масштаб и позиция камеры сохраняются** (без «телепорта» в центр)
- **Расчёт:** производительность линии (min по цепочке), узкое место, энергия, стоимость станков / подачи / конвейеров, установка
- **PDF-отчёт** (компактный, один лист при возможности) с кириллицей
- **postMessage** родителю после сохранения (`leskom:configurator:saved`)

## Типы оборудования (`catalogType`)

| Тип | Вкладка | На поле | В расчёте |
|-----|---------|---------|-----------|
| `machine` | Станки | Да | Таблица станков, итог по станкам |
| `log_feed` | Подача бревна | Да | Отдельный блок, если размещено |
| `conveyor` | Конвейеры | Нет (только на связи) | Конвейеры на связях + итог |
| `equipment_complex` | Комплексы и линии | Разворачивается в станки + связи | Как станки |

Классификация: `catalog-meta.js` (имя, slug, legacy id, шаблоны линий) и API `configurator_catalog.py` на бэкенде.

## Запуск

### Локально (модуль)

```bash
cd Machine-Tool-Calculator
npx serve .
```

Откройте `http://localhost:3000`. При `file://` API и SQLite недоступны — только `equipment-data.js`.

### На сайте leskomphp

Статика копируется в `static/configurator/`:

```bash
npm run copy:configurator
npm run build
```

Конфигуратор: `/configurator/?embed=1&projectId=<uuid>` в iframe из `/cabinet/configurator`.

API: `cd api && uvicorn app.main:app --reload --port 8000`

Подробно: [docs/CONFIGURATOR-INTEGRATION.md](docs/CONFIGURATOR-INTEGRATION.md)

## Верхняя панель

| Элемент | Назначение |
|---------|------------|
| **Название линии** | Заголовок сохранённой конфигурации |
| **Каталог** | Боковая панель оборудования |
| **Очистить** | Убрать всё с поля |
| **Связать** | Режим создания связей |
| **Как пользоваться** | Инструкция (краткая для клиента; расширенная для менеджера и выше) |
| **Клетка** | Шаг сетки |
| **− / + / ⊙** | Масштаб и центр |
| **Рассчитать** | Расчёт, просмотр, PDF |

Меню «Проект» (загрузка черновика, экспорт/импорт JSON) **убрано** — работа только через сохранение на сайте. Раздел для менеджеров в инструкции показывается при роли `manager` и выше или праве `configs.manage` (параметр `staff=1` в URL iframe).

## Управление

| Действие | Как |
|----------|-----|
| Станок / подача бревна | Перетащить из каталога |
| Комплекс / лесопильная линия | Вкладка «Комплексы и линии» → перетащить на поле (развернётся в станки) |
| Связь | «Связать» → источник → приёмник, или **Shift** + два клика (без сдвига мыши) |
| Конвейер на связи | В модальном окне; или **Без конвейера** |
| Изменить / удалить связь | **ПКМ** по линии |
| Выделение области | ЛКМ на пустом месте — протянуть рамку |
| Группа | Перетащить любой объект из выделения; **Delete** — удалить выделенное |
| Один станок | Перетащить карточку; **×** на карточке — удалить |
| Зум | **Ctrl** + колесо или кнопки **− / +** |
| Сдвиг поля | **Alt** / СКМ / ПКМ / ЛКМ по пустому месту (не в режиме «Связать») |
| Центр | Кнопка **⊙** |

В режиме **«Связать»** или при **Shift** рамка выделения не мешает установке связей. Перетаскивание станка не считается кликом для связи. При нескольких связях из одного станка линии рисуются **раздельно**, с подписями конвейера на длинных участках.

## Сохранение и синхронизация

| Режим | Поведение |
|-------|-----------|
| Один редактор | Автосохранение на сервер; лёгкий опрос ~24 с |
| Два окна / два пользователя | Опрос ~4,5 с; правки подтягиваются **без затемнения** поля |
| Во время drag / правки | Подгрузка с сервера ждёт, пока отпустите мышь и закончится автосохранение |
| Камера | При подгрузке с сервера остаётся ваш zoom и смещение |

API (cookie-сессия):

| Метод | Путь | Назначение |
|-------|------|------------|
| GET | `/api/v1/configurator/projects/{id}` | Полная схема |
| GET | `/api/v1/configurator/projects/{id}/sync?session_id=` | Статус и `others_editing` |
| POST | `/api/v1/configurator/projects/{id}/presence` | Heartbeat вкладки (не меняет время изменения схемы) |
| POST | `/api/v1/configurator/projects` | Создать / обновить |

Параметр iframe `staff=1` — расширенная инструкция «Как пользоваться» для менеджеров.

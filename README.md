# Онлайн-сборщик производственного цеха

Standalone версия - работает без сервера, просто откройте `index.html`!

## 🚀 Быстрый старт

1. **Копирование базы данных:**
   ```
   python copy_db.py
   ```
   Это скопирует базу данных `factory.db` из основного проекта в папку `data/`

2. **Откройте `index.html`** двойным кликом - всё работает!
   
   Приложение автоматически загрузит данные из `data/factory.db` используя SQL.js (WebAssembly версия SQLite)

## 📁 Структура проекта

```
server popa/
├── index.html          # Главная страница
├── css/
│   └── style.css      # Стили
├── js/
│   ├── main.js        # Основная логика
│   ├── connections.js # Управление связями
│   └── equipment-data.js # Данные оборудования (резервный вариант)
├── data/
│   └── factory.db     # База данных SQLite (создается при копировании)
├── images/
│   └── equipment/     # Картинки станков (создается при экспорте)
├── copy_db.py         # Скрипт копирования базы данных
└── export_from_sqlite.py # Скрипт экспорта данных (альтернативный способ)
```

## 🔧 Обновление данных

Если нужно обновить данные из базы:

1. Убедитесь, что база данных обновлена (запустите `load_catalog_to_db.py` в основном проекте)
2. Запустите: `python copy_db.py` (копирует базу данных)
3. Обновите страницу в браузере (F5)

**Альтернативный способ (с экспортом в JS):**
1. Запустите: `python export_from_sqlite.py` (создает `equipment-data.js`)
2. Обновите страницу в браузере (F5)

## 📝 Примечания

- **ВАЖНО**: При открытии `index.html` через `file://` протокол (двойной клик) SQLite база данных **не будет загружаться** из-за ограничений безопасности браузера (CORS). В этом случае автоматически используется резервный вариант - встроенные данные из `equipment-data.js`
- Для работы с SQLite базой данных рекомендуется использовать локальный веб-сервер (например, `python -m http.server` или Live Server в VS Code)
- База данных SQLite читается напрямую в браузере через SQL.js (WebAssembly) при работе через HTTP/HTTPS
- Если база данных недоступна, используется резервный вариант - встроенные данные из `equipment-data.js`
- Картинки должны быть в папке `images/equipment/` (копируются при экспорте)
- Не требует установки дополнительных библиотек для работы в браузере
- SQL.js загружается с CDN автоматически
